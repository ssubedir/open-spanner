Open Spanner can run as a single container with SQLite or as an app plus Postgres stack with Docker Compose.

Use Docker Run when you want the smallest local instance. Use Docker Compose when you want the service running against Postgres.

## Option 1: Docker Run

This starts one Open Spanner container using SQLite storage persisted in a Docker volume.

```sh
git clone https://github.com/ssubedir/open-spanner.git
cd open-spanner
docker build -t open-spanner:local .
docker volume create open-spanner-data
docker run --detach \
  --name open-spanner \
  --publish 18081:18081 \
  --volume open-spanner-data:/data \
  open-spanner:local
```

Open the dashboard at `http://localhost:18081/register`.

Stop and remove the container:

```sh
docker rm -f open-spanner
```

The data stays in the `open-spanner-data` volume until you remove that volume.

## Option 2: Docker Compose

This starts Open Spanner and Postgres together. Postgres is private to the Compose network and is not published on host port `5432`.

```sh
git clone https://github.com/ssubedir/open-spanner.git
cd open-spanner
docker compose -f docker-compose.app.yml up -d --build
```

Open the dashboard at `http://localhost:18081/register`.

To change the published app port or database password, create a `.env` file next to `docker-compose.app.yml`:

```sh
OPEN_SPANNER_HTTP_PORT=8080
OPEN_SPANNER_POSTGRES_PASSWORD=change-me
```

Then start the stack:

```sh
docker compose -f docker-compose.app.yml up -d --build
```

Stop the stack:

```sh
docker compose -f docker-compose.app.yml down
```

Remove the Postgres data volume as well:

```sh
docker compose -f docker-compose.app.yml down -v
```

## Option 3: Run From Source

Use Task when you want to run the service directly from the repository.

<Tabs items={["npm", "macOS/Linux", "Windows"]}>
<Tab value="Windows">

  ```sh
  winget install Task.Task

  </Tab>
<Tab value="macOS/Linux">

  brew install go-task/tap/go-task
  ```

  </Tab>

  ```sh
  npm install -g @go-task/cli
  ```

  </Tab>

Run with SQLite:

```sh
git clone https://github.com/ssubedir/open-spanner.git
cd open-spanner
task run:sqlite
```

## Create An API Key

After registering a dashboard user, open `/api-keys`, create a key for your service, and copy it immediately.

```sh
API_KEY="osp_..."
```

## Create A Meter

```sh
curl -X POST http://localhost:18081/v1/meters \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "name": "api_requests",
    "description": "API requests served",
    "unit": "request",
    "aggregation": "sum",
    "event_retention_days": 90
  }'
```

## Send Usage

```sh
curl -X POST http://localhost:18081/v1/usages \
  -H "Authorization: Bearer $API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "subject": "org_123",
    "meter": "api_requests",
    "quantity": 1,
    "idempotency_key": "usage_001",
    "metadata": {
      "region": "us-east",
      "plan": "pro"
    }
  }'
```
