---
title: Your First Query
description: Query bucketed usage for a meter.
---

Usage queries return buckets. A bucket is a time window with a meter aggregation applied.

This query returns daily `api_requests` usage across every subject:

```sh
curl "http://localhost:18081/v1/usages?meter=api_requests&from=2026-06-01T00:00:00Z&to=2026-06-10T00:00:00Z&bucket_size=day" \
  -H "Authorization: Bearer $OPEN_SPANNER_API_KEY"
```

Add `subject=org_123` when you want one subject instead of the full meter:

```sh
curl "http://localhost:18081/v1/usages?subject=org_123&meter=api_requests&from=2026-06-01T00:00:00Z&to=2026-06-10T00:00:00Z&bucket_size=day" \
  -H "Authorization: Bearer $OPEN_SPANNER_API_KEY"
```

Use `group_by=subject` when you want one bucket row per subject in the same response. For richer filters or grouping, use `POST /v1/usages/search` or the dashboard usage page.
