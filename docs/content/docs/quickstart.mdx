## Run The API

Use SQLite when you want local persistence without a separate database:

```sh
task run:sqlite
```

The server starts the API, embedded dashboard, and Swagger UI from the same process.

## Create A Meter

Meters define what is being measured and how usage should be aggregated.

```sh
curl -X POST http://localhost:18081/v1/meters \
  -H "Content-Type: application/json" \
  -d '{
    "name": "api_requests",
    "description": "API requests served",
    "unit": "request",
    "aggregation": "sum",
    "event_retention_days": 90
  }'
```

## Record Usage

```sh
curl -X POST http://localhost:18081/v1/usages \
  -H "Content-Type: application/json" \
  -d '{
    "subject": "org_123",
    "meter": "api_requests",
    "quantity": 1,
    "idempotency_key": "usage_001",
    "metadata": {
      "region": "us-east",
      "plan": "pro"
    }
  }'
```

## Query Usage

```sh
curl "http://localhost:18081/v1/usages?subject=org_123&meter=api_requests&from=2026-06-01T00:00:00Z&to=2026-06-10T00:00:00Z&bucket_size=day"
```

For richer filtering, use `POST /v1/usages/search` or the dashboard usage page.
