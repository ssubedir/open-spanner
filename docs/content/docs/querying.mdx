---
title: Querying Usage
description: Query bucketed usage with time windows, grouping, and advanced filters.
---

Usage queries return buckets over a requested time window. The query shape is meant to support simple API calls and rich UI filtering without accepting SQL from clients.

## Basic Query

```sh
curl "http://localhost:18081/v1/usages?subject=org_123&meter=api_requests&from=2026-06-01T00:00:00Z&to=2026-06-10T00:00:00Z&bucket_size=day"
```

## Advanced Query

Use `POST /v1/usages/search` for nested filter groups.

```json
{
  "from": "2026-06-01T00:00:00Z",
  "to": "2026-06-10T00:00:00Z",
  "bucket_size": "day",
  "group_by": "region",
  "limit": 500,
  "filter": {
    "type": "group",
    "op": "and",
    "rules": [
      {
        "type": "condition",
        "field": "subject",
        "op": "eq",
        "value": "org_123"
      },
      {
        "type": "condition",
        "field": "meter",
        "op": "eq",
        "value": "api_requests"
      }
    ]
  }
}
```

The dashboard uses this advanced filter model through `react-querybuilder`.

## Result Shape

Each bucket includes the time range, aggregation value, meter, unit, and optional group value.
