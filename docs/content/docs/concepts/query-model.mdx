title: Query Model
description: How usage events become buckets.
---

Open Spanner stores usage events and returns aggregates over time windows.

## Buckets

A bucket has:

- Start and end time
- Meter
- Unit
- Aggregated value
- Optional group value

The meter's aggregation mode decides how the value is calculated.

## Basic Query

Use query parameters when you know the direct filters:

```sh
curl "http://localhost:18081/v1/usages?subject=org_123&meter=api_requests&from=2026-06-01T00:00:00Z&to=2026-06-10T00:00:00Z&bucket_size=day" \
  -H "Authorization: Bearer $OPEN_SPANNER_API_KEY"
```

## Advanced Query

Use `POST /v1/usages/search` when you need nested filter groups or grouping:

```json
{
  "from": "2026-06-01T00:00:00Z",
  "to": "2026-06-10T00:00:00Z",
  "bucket_size": "day",
  "group_by": "region",
  "filter": {
    "type": "group",
    "op": "and",
    "rules": [
      {
        "type": "condition",
        "field": "subject",
        "op": "eq",
        "value": "org_123"
      },
      {
        "type": "condition",
        "field": "meter",
        "op": "eq",
        "value": "api_requests"
      }
    ]
  }
}
```
